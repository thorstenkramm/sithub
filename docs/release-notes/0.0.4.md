# Release notes for version 0.0.4

## 🎉 New Features

- **Real-time booking updates**: bookings and cancellations made by other users propagate
  to every open client in seconds across the tile, weekly table, and floor plan views,
  without requiring a manual reload; reconnection after a network outage reconciles
  missed events and multiple tabs stay in sync
- **Favorites virtual room**: when you have at least one favorite item, a dedicated
  "Favorites" tile appears as the first tile on the area overview and behaves like any
  other room with drill-down, free/busy indicators, and identical interaction; heart
  icons on tile, table, and floor plan let you remove a favorite from any view
- **Booker avatars on item tiles**: every booked item tile in both day and week modes
  now shows the booker's avatar (or initials when no image is available) with the full
  display name on hover or tap; matches the floor plan presentation
- **Inline colleague booking**: the colleague dropdown is always rendered and always
  enabled — picking a colleague books on their behalf, leaving it empty books for
  yourself, eliminating the previous "Book for a colleague" radio/checkbox
- **Compact single-line booking controls**: day/week toggle, date or week selector,
  floor plan button, equipment filter, and colleague selector now sit on a single row
  on wide viewports; the booking controls card is visibly shorter, leaving more room
  for tiles
- **Login page rebrand**: Entra ID is now the primary sign-in option with the official
  Microsoft Entra ID icon; the local username/password form is hidden behind a "more
  login options" link and reveals on demand; servers without Entra ID configured show
  the local form by default so nobody is locked out
- **New SitHub brand logo**: full vertical logo on the login page and a compact
  horizontal variant in the application header; the app-bar is now a light surface
  with subtle elevation to let the brand color shine
- **New favicon**: redrawn from the new brand logo so browser tabs match the rest of
  the experience
- **Equipment filter on the weekly table view**: typing a keyword in the equipment
  filter dims rows that don't match while keeping matching desks at full opacity; the
  same input drives the card and table views and saved-filter persistence is unchanged
- **Optional area drill-down on floor plans**: a new toggle on the floor-plan booking
  view lets large-screen users skip the room drill-down and book items directly;
  default off on large screens, on for small, persisted per device

## 🔩 Improvements

- Cancellation in week mode now happens through the booked-by-me checkbox itself — the
  separate red cancel-X icon is gone and the existing confirmation dialog still gates
  the action
- Booker name on week-mode tiles is sourced from the booking record (not the current
  user), so bookings made on behalf of a colleague correctly show the colleague's
  identity everywhere
- Folded week tile no longer shows redundant booker-name text; the name appears in
  the avatar tooltip on hover or tap
- Weekly-table booking-cancel popover is wide enough that the "Cancel booking" button
  is fully visible regardless of viewport height
- Equipment filter info icon now sits immediately adjacent to the filter input on
  both the item-groups and items pages
- Floor plan editor canvas now grows vertically on zoom-in so the full image height is
  visible without vertical scrolling
- Areas configuration example documents that the areas config file must live inside
  `data_dir`
- Reusable FloorPlanButton component gives the floor-plan button consistent sizing on
  every page it appears

## 🐞 Bug Fixes

- Equipment filter on the item-groups view did not reset when cleared via the input's
  X icon, leaving tiles blurred until a page reload
- Equipment filter save icon was rendering as an invisible button (saved filters could
  not be created); the icon now displays correctly using a registered Vuetify alias
- Equipment filter input width no longer jitters when filter content is typed or
  cleared — the save/delete action button reserves its space in both states
- Server now refuses to start when the areas configuration contains duplicate item
  identifiers, with a clear error naming the duplicate and its locations, instead of
  failing later at booking time

## 🔌 API

- New public endpoint `GET /api/v1/auth/providers` reports which authentication
  providers (Entra ID, local) are configured on the server; used by the login page to
  gate the Entra ID button and the "more login options" toggle

## ⬆️ Upgrade Notes

- No database migrations required. Rebuild and redeploy as usual.
- No configuration changes required. Entra ID and local-auth setups continue to work
  unchanged; the new `/api/v1/auth/providers` endpoint introspects the existing
  `[entraid]` configuration.
- The application header background is now a light surface; if you have any
  deployment-specific CSS overrides targeting the old `color="primary"` app-bar, you
  may want to revisit them.
