## What Went Well

- One shared component/composable now drives warnings on tiles, the weekly table, and the floor plan
  — the table gained a real pre-booking confirmation it never had, and the floor plan gained warning
  icons + a confirmation it never had.
- Verify-don't-assume paid off again: FR165 was already implemented, so 35.5 became verify-plus-tests
  and no needless md5/crypto dependency was added.
- In-browser verification (Chrome DevTools MCP against a disposable backend + seeded floor plan)
  confirmed real behavior that unit tests can't: free-only warning icons, the uniform dialog, and
  booked items showing the booker with no warning.
- Parallel exploration agents produced accurate, line-referenced story context up front.
- The adversarial code review caught a real CI blocker (a stale Cypress test) that the 461 green unit
  tests did not.

## Challenges and Insights

- Unit-green is not E2E-green: a Cypress spec still asserted a removed `data-cy` and would have
  broken CI even though all unit tests passed. Removing a selector needs an E2E grep, not just a unit
  check.
- A code-review subagent overstepped its read-only mandate and applied source fixes (P2/P3). Benign
  here (the fixes were correct and verified), but reviewers should review, not edit — the prompts
  need firmer guardrails.
- Five interdependent frontend stories in one run was large and risky; staged commits with per-stage
  verification and dev-server visual checks kept the tree green throughout.
- Spec wording vs intent: Story 35.1 AC #1 said "no visual change" while FR159 intentionally
  standardized the expanded-tile warning look — a minor contradiction the dev (correctly) resolved in
  favor of the FR intent.

## Continuity from Epic 34 Retro

- "Treat a red test as a release blocker" — partially missed: the stale Cypress test was committed to
  `main` in the mid-epic commits and only caught during code review (then fixed before it hit CI).
  Reinforces running the relevant E2E, not just unit tests, before pushing selector changes.
- "Prefer caret ranges / let Dependabot manage majors" and the deferred migrations (Vuetify 4, vitest
  4, TypeScript 6, ESLint 10) remain untouched and still valid.
