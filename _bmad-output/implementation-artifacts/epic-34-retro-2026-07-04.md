## Epic Summary

Epic 34 hardened SitHub against the findings of the AI-assisted security review, implementing
FR153–FR158 across five stories. All work shipped in a single commit (`c06812e`), fast-forwarded to
`main` and pushed.

### Delivery metrics

- Stories: 5 / 5 done (100%)
  - 34.1 HTTP security response headers (Echo `Secure` middleware)
  - 34.2 `force_secure_cookies` config for TLS-terminating reverse proxies
  - 34.3 slow-client server timeouts + 2 MB request body limit
  - 34.4 pinned npm dependency versions + Dependabot
  - 34.5 security regression test coverage
- Requirements delivered: FR153–FR158 (6 functional requirements)
- Net-new security tests: ~11 (headers, body-limit, cookie force-secure cases, rate-limit 429,
  avatar + floor-plan path traversal, invalid/expired session, booking-note XSS escaping)

### Quality and security outcomes

- npm vulnerabilities: 13 -> 0
- Trivy Go CVEs (golang.org/x/net, x/sys, x/crypto, x/text): resolved -> 0
- Backend Go test suite: all passing; frontend unit tests: 444 / 444 (6 long-standing `ItemsView`
  week-mode failures fixed during the epic)
- Production incidents: 0
- CI hardening added: two-stage Trivy scan (frontend npm / backend Go + secrets) with a shared,
  cached vulnerability DB

## What Went Well

- Verify-don't-assume: the review flagged 7 test gaps; analysis showed 3 were already covered
  (floor-plan traversal, admin self-delete, SameSite=Lax). Existing coverage was documented instead
  of duplicated.
- Story preparation caught three high-impact integration gotchas before implementation: the Echo
  middleware package pulling in `golang.org/x/time/rate` (missing from `go.sum`), a strict CSP that
  would break Google Fonts, and a global 2 MB body limit that would reject the 4 MB avatar upload.
- Security posture improved measurably: dependency vulnerabilities driven to zero on both the npm
  and Go sides, plus a durable CI scan to prevent regressions.

## Challenges and Insights

- Pinning surfaced latent debt: Story 34.4's version pinning exposed pre-existing vulnerabilities,
  and the exact pin `@vitest/coverage-v8@3.2.4` actively blocked a security patch until it was
  loosened to a caret range.
- Silent test rot: 6 `ItemsView` week-mode tests had been failing on `main` before the epic and went
  unnoticed; they were fixed via explicit date-state seeding during the epic.
- Tooling drift: the jscpd flag documented as `--exitCode` was actually `--exit-code`; found and
  corrected across CI, `run-all-tests.sh`, and the rules docs.
- Scan scoping: Trivy flagged a local, gitignored SSH private key under `private/` (not a repo leak);
  the scan commands needed `--skip-dirs private`.
- Honest gap: Story 34.1 Task 4 (live-browser CSP smoke test) was not performed — verified by static
  analysis only.
